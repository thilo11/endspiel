# Endspiel — NNUE Training Guide

This document describes the tools used to generate data, train a net, and run it
against another engine. Decisions about whether a candidate net is good enough
to ship belong in the PR that proposes the promotion, not here.

## Prerequisites

- [Bullet](https://github.com/jw1912/bullet) checked out at `../bullet` (sibling of this repo) — pipeline scripts use its `bullet-utils` binary. The `train/` crate itself pulls `bullet_lib` via a pinned git rev and does **not** need the checkout.
- CUDA toolkit installed. The trainer is built against bullet's CUDA backend; there is no CPU fallback. `train/.cargo/config.toml` defaults `CUDA_PATH` to `/usr/local/cuda`; if your toolkit lives elsewhere, `export CUDA_PATH=/your/path` and it will override the default.
- [fastchess](https://github.com/Disservin/fastchess) for match testing.

Build the `bullet-utils` helper once (used by the pipeline scripts):

```bash
cd ../bullet
cargo build --release -p bullet-utils
```

## Architecture

Current net: **HalfKP 704×32→768×2→1 SCReLU**

- Input: 704 HalfKP features × 32 per-square king buckets = 22,528 input rows
- Two hidden layers of width 768, SCReLU activation
- Bullet input format: `ChessBucketsMergedKingsMirrored` with identity bucket layout
- Net file: `crates/chess-nnue/nets/default.nnue` (~33 MB)

## Training Loop

One round of the loop:

```
1. Generate self-play data  →  data/new.bin
2. Mix new data with archive →  data/data_train_next.bin
3. Train from scratch
4. Select best checkpoint
5. Test the candidate (self-play and/or vs SF)
6. If promoted: copy net, refit WDL, rebuild
7. Merge new data into the archive
```

Data is merged into the archive regardless of whether the net was promoted —
positions generated by the current net are valid training material for
future rounds.

## Data Generation

```bash
cargo build --release -p chess-datagen

target/release/chess-datagen \
  --games 2000000 \
  --depth 16 \
  --threads 32 \
  --hash 16 \
  --syzygy assets/syzygy \
  --output data/new.bin
```

| Flag | Default | Description |
|------|---------|-------------|
| `--games N` | 2000000 | Self-play games to generate |
| `--depth N` | 16 | Search depth per move |
| `--threads N` | 32 | Parallel game threads |
| `--hash MB` | 16 | TT size per thread (scale with depth: 8–16 for ≤10, 32–64 for 12, 256+ for ≥16) |
| `--output PATH` | `data/data_new.bin` | Output file (append mode); the pipeline overrides this to `data/new.bin` |
| `--nnue bool` | true | Use NNUE eval; set false for round 1 if no net exists yet |
| `--syzygy PATH` | — | Syzygy directory for exact endgame labels |
| `--book FILE` | — | Polyglot `.bin` opening book |
| `--start-fens FILE` | — | Newline-separated FEN list to seed each game (overrides the random-opening prefix) |
| `--sequential-fens` | off | With `--start-fens`, consume each FEN once in order instead of sampling randomly |

Default opening behaviour: 8 fully random legal moves per game (no book), then
recording starts from the first engine ply. `--start-fens` is the way to target
specific phases (e.g. middlegame, endgame studies) — pair with
`--sequential-fens` for reproducible coverage of a fixed list.

## Data Preparation

### Check integrity (after interrupted runs)

```bash
python3 -c "import os; print(os.path.getsize('data/new.bin') % 32, 'leftover bytes')"
# If non-zero:
truncate -s $(($(stat -c%s data/new.bin) - $(stat -c%s data/new.bin) % 32)) data/new.bin
```

### Count positions

```bash
stat -c%s data/new.bin | awk '{printf "%.1fM positions\n", $1/32/1e6}'
```

### Prepare the archive

`next_round_pipeline.sh` consumes a pre-processed archive (`archive_ready.bin`)
that has already been deduped, cleaned, and shuffled. Produce it with:

```bash
bash scripts/prepare_archive.sh --input data/archive/archive.bin
```

Defaults: dedup=off, draw-ratio=0.60, drop-king-only=true, drop-high-abs-draws=0.

Dedup is off by default because the in-memory HashSet is RAM-bound
(~24 B × unique positions) and will OOM on multi-billion-position archives.
Pass `--dedup` when the archive fits in RAM.

After each round, append new data to the raw archive and re-run:

```bash
cat data/new.bin >> data/archive/archive.bin
bash scripts/prepare_archive.sh --input data/archive/archive.bin
```

## Training

### Full pipeline

```bash
bash scripts/next_round_pipeline.sh --from-scratch --run-mix
```

This runs: clean fresh data → shuffle → interleave with the prepared archive
→ train → checkpoint summary.

The archive must be pre-processed with `prepare_archive.sh` first.

Fresh data defaults: dedup=on, draw-ratio=0.60, drop-king-only=true.

Pipeline flags:

| Flag | Description |
|------|-------------|
| `--from-scratch` | Train from random init |
| `--run-mix` | Clean + interleave `data/new.bin` with archive before training |
| `--run-datagen` | Generate new `data/new.bin` first |
| `--games N` | Override game count for this round |
| `--superbatches N` | Override superbatch count (from-scratch default: 250) |
| `--lr F` | Override learning rate (from-scratch default: 0.001) |
| `--wdl F` | Override WDL mixing (from-scratch default: 0.75) |
| `--skip-train` | Data preparation only, no training |
| `--new-ratio F` | Mix ratio for new data (default: proportional to file size) |

### Manual trainer invocation

```bash
cd train
cargo run --release --bin train -- \
  --data ../data/data_train_next.bin \
  --superbatches 200 \
  --epochs 3 \
  --wdl 0.75 \
  --lr 0.001 \
  --name round_N
```

Trainer flags:

| Flag | Default | Description |
|------|---------|-------------|
| `--data PATH` | `../data.bin` | Training data |
| `--superbatches N` | 60 | Total superbatches |
| `--epochs N` | 1 | Dataset passes per superbatch |
| `--lr F` | 0.001 | Initial LR (cosine decay to `lr × 0.3^5`) |
| `--wdl F` | 0.75 | WDL mixing proportion |
| `--save-rate N` | 10 | Save every N superbatches |
| `--name ID` | `endspiel` | Checkpoint name prefix |
| `--output DIR` | `checkpoints` | Output directory |
| `--init-net PATH` | — | Warm-start from an existing net |
| `--finetune` | off | Warm-start preset: lr=0.00001, superbatches=8, wdl=0.5, bps=8 (requires `--init-net`) |
| `--resume DIR` | — | Resume from checkpoint directory |
| `--start N` | 1 | Starting superbatch (for resume) |
| `--bps N` | auto | Batches per superbatch (overrides auto-computed value) |

### Checkpoint selection

```bash
cd train
python3 best_checkpoint.py
```

Checkpoints are saved under `train/checkpoints/<name>-N/quantised.bin`.

## Testing a Candidate Net

### Load a net without rebuilding

```
setoption name EvalFile value /abs/path/to/quantised.bin
```

### Sanity check before a match

fastchess silently ignores an invalid `EvalFile` and falls back to the embedded
net without warning. Verify both engines load the intended net:

```bash
for NET in path/to/net_a.bin path/to/net_b.bin; do
  echo "=== $NET ==="
  printf "uci\nsetoption name EvalFile value %s\nisready\nposition startpos\ngo movetime 100\n" "$NET" \
    | target/release/endspiel | grep -E "^info|^bestmove"
done
```

The two runs must produce different `info score` lines. Identical lines mean
one path is wrong.

### Self-play match

```bash
fastchess \
  -engine cmd=target/release/endspiel name=candidate \
          option.EvalFile=/abs/path/train/checkpoints/<name>-N/quantised.bin \
          option.Hash=64 option.Threads=1 \
  -engine cmd=target/release/endspiel name=baseline \
          option.Hash=64 option.Threads=1 \
  -openings file=assets/openings.epd format=epd order=random \
  -each tc=10+0.1 \
  -rounds 250 -concurrency 8 -recover
```

Required options (omitting either causes games to hang or OOM):
- `option.Hash=64` — default is 256 MB; 8 concurrent games × 2 engines = OOM without this
- `option.Threads=1` — default is 16; 8 concurrent games × 16 threads = CPU stall

### Match vs Stockfish

```bash
fastchess \
  -engine cmd=target/release/endspiel \
          option.Hash=64 option.Threads=1 name=endspiel \
  -engine cmd=/usr/games/stockfish \
          option.UCI_LimitStrength=true option.UCI_Elo=3190 \
          option.Hash=64 option.Threads=1 name=sf3190 \
  -openings file=assets/openings.epd format=epd order=random \
  -each tc=10+0.1 -rounds 200 -concurrency 8 -recover
```

## Swapping the Embedded Net

The default net is compiled into the binary via `include_bytes!`. Copying a
file in place has no effect until the engine is rebuilt.

```bash
cp train/checkpoints/<name>-N/quantised.bin crates/chess-nnue/nets/default.nnue

# Refit WDL normalization for the new net:
target/release/wdl-fit --input data/new.bin
# → copy the printed WDL_A / WDL_B into crates/chess-uci/src/handler.rs

cargo build --release   # REQUIRED — net is embedded at compile time
```
